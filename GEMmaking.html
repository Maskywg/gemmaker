3. 實際創建流程

四個簡單步驟，從零打造你的專屬大師。

Step 1. 賦予身份與命名

給你的 Gem 一個清晰的名字和簡短描述（例如：「SEO 爆款文章寫手」）。這有助於你在多個 Gem 中快速找到它。

1
Name: 資深 Python 架構師
Description: 專精於編寫高效能且符合 PEP 8 規範的 Python 程式碼。

Step 2. 撰寫核心指令 (Instructions)

這是最關鍵的一步。你需要詳細規定它的行為模式、語氣、以及必須遵守的規則。建議使用清晰的結構（如 Markdown 或 YAML 格式）。

4. Gem 的整理術

隨著創建的 Gem 越來越多，掌握良好的命名與管理技巧能大幅提升您的工作效率。

🏷️

標籤化命名法

在 Gem 的名稱前方加上統一的括號標籤或 Emoji（例如：[行銷] SEO 寫手 或 🐍 Python 專家），讓側邊欄清單一目了然。

📌

善用釘選功能

將您每天都會用到的 3-5 個核心 Gem 釘選在 Gemini 側邊欄的最上方，確保每次需要處理專案時都能一鍵喚醒您的專屬數位大腦。

查看釘選教學
🔄

版本與描述紀錄

不斷優化指令是常態。建議在 Description 欄位註記版號（如 v1.2），或將指令備份在文件裡，記錄優化過程以防把原本穩定的 Gem 改壞。

🔗

分享與連結

創建好的 Gem 可以生成專屬分享連結傳給同事或朋友。善用這項功能，讓您調教出的強大 AI 助理直接成為整個團隊的生產力外掛。

查看分享教學
